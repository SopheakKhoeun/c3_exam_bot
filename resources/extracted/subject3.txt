Part A вҖ” The scenario
Toul Kork Mini-Mart is a small family shop in Phnom Penh selling rice, drinks, and everyday goods. Until now the owner tracked everything in a paper notebook: which products are in stock, who ordered what, and which orders are paid.
The shop hired your team to build the backend API. A previous developer already set up the Express project, the routes, and part of the database schema вҖ” your job is to finish it in 2 hours.
The shop's rules:
вҖў  A customer can order a product only if enough stock is available.
вҖў  The total price is always quantity Г— product price вҖ” calculated by the server, never sent by the client.
вҖў  A new order starts as PENDING; only pending orders can be cancelled вҖ” the stock goes back on the shelf.
What is provided (do not change)
вҖў  Express app, routes, controllers, error handling вҖ” all working
вҖў  Customer and Product models in prisma/schema.prisma
вҖў  Seed data (npm run seed)
вҖў  requests.http to test every task
What you must do
The tasks in Part C вҖ” marked TODO in the code. Only these files are yours to edit:
вҖў  prisma/schema.prisma
вҖў  src/repositories/product.repository.js
вҖў  src/repositories/customer.repository.js
вҖў  src/services/order.service.js
Setup steps (do this first, ~10 minutes)
1. Fork the starter repository to your own GitHub account, then clone your fork. Name your repo C3_<StudentID>_<YourName>.
2. npm install
3. Create an empty PostgreSQL database named storedb
4. cp .env.example .env and edit DATABASE_URL
5. Start the server: npm run dev вҖ” you should see "Server running on http://localhost:3000"
(Migration and seed come later вҖ” they are part of task S5.)

S7 вҖ” Server-side business components (35 pts)
Implement the TODO functions in src/services/order.service.js. All business rules are written above each function.
S7.1 вҖ” Place an order (25 pts)
POST  /api/orders

{ "customerId": 3, "productId": 3, "quantity": 2 }

вҖў  400 if quantity is not a whole number greater than 0
вҖў  404 if customer or product does not exist
вҖў  409 if the product does not have enough stock left
вҖў  totalPrice = quantity Г— price вҖ” calculated by the server, never taken from the request
вҖў  creates the order (status PENDING by default) and decreases stock by quantity in one transaction (prisma.$transaction)
S7.2 вҖ” Cancel an order (10 pts)
PUT  /api/orders/2/cancel

вҖў  404 if the order does not exist
вҖў  409 if the order is not PENDING (a PAID order cannot be cancelled)
вҖў  sets status = CANCELLED and increases the productвҖҷs stock by the orderвҖҷs quantity
